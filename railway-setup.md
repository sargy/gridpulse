# Railway Deployment Setup

This document describes how to recreate the GridPulse Railway deployment from scratch.

## Architecture

```
┌─────────────────────────────────────────────────────┐
│  Railway Project: gridpulse                         │
│                                                     │
│  ┌───────────┐    ┌───────────┐    ┌──────────────┐ │
│  │  frontend  │───▶│  backend   │───▶│   MongoDB    │ │
│  │  (nginx)   │    │ (Spring)   │    │  (managed)   │ │
│  │  port 80   │    │  port 8080 │    │  port 27017  │ │
│  └───────────┘    └───────────┘    └──────────────┘ │
│       │                                              │
│  public domain         internal network only         │
└─────────────────────────────────────────────────────┘
```

- **Frontend** serves the React SPA and proxies `/api/*` to the backend over Railway's private network
- **Backend** connects to MongoDB over the private network
- **MongoDB** is a Railway managed database (not publicly exposed in production)

## Prerequisites

- [Railway CLI](https://docs.railway.com/guides/cli) installed (`npm i -g @railway/cli`)
- Railway account with GitHub app installed on the `sargy/gridpulse` repo
- Logged in: `railway login`

## Step-by-Step Setup

### 1. Create the project

```bash
railway init --name gridpulse
```

### 2. Add MongoDB

```bash
railway add --database mongo
```

This creates a MongoDB service with auto-generated credentials. Railway sets these variables automatically:

| Variable | Value |
|----------|-------|
| `MONGO_URL` | `mongodb://<MONGO_USER>:<MONGO_PASSWORD>@mongodb.railway.internal:27017` |
| `MONGO_PUBLIC_URL` | `mongodb://<MONGO_USER>:<MONGO_PASSWORD>@<PROXY_HOST>:<PROXY_PORT>` |
| `MONGOUSER` | `<auto-generated>` |
| `MONGOPASSWORD` | `<auto-generated — SECRET>` |
| `MONGOHOST` | `mongodb.railway.internal` |
| `MONGOPORT` | `27017` |

### 3. Create the backend service

```bash
railway add --service "backend"
```

#### Connect GitHub repo (via API — CLI doesn't support root directory)

```bash
RAILWAY_TOKEN=$(python3 -c "import json; d=json.load(open('$HOME/.railway/config.json')); print(d['user']['token'])")

# Connect repo
curl -s -X POST https://backboard.railway.app/graphql/v2 \
  -H "Authorization: Bearer $RAILWAY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "mutation { serviceConnect(id: \"<BACKEND_SERVICE_ID>\", input: { repo: \"sargy/gridpulse\", branch: \"feature/full-stack-rewrite\" }) { id } }"
  }'

# Set root directory and Dockerfile path
curl -s -X POST https://backboard.railway.app/graphql/v2 \
  -H "Authorization: Bearer $RAILWAY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "mutation { serviceInstanceUpdate(serviceId: \"<BACKEND_SERVICE_ID>\", environmentId: \"<ENV_ID>\", input: { rootDirectory: \"/backend\", dockerfilePath: \"Dockerfile\" }) }"
  }'
```

#### Set environment variables

```bash
railway service link backend
railway variable set SPRING_DATA_MONGODB_URI='${{MongoDB.MONGO_URL}}/gridpulse?authSource=admin'
```

| Variable | Value | Notes |
|----------|-------|-------|
| `SPRING_DATA_MONGODB_URI` | `${{MongoDB.MONGO_URL}}/gridpulse?authSource=admin` | Railway reference variable — resolves to `mongodb://<USER>:<SECRET_PASSWORD>@mongodb.railway.internal:27017/gridpulse?authSource=admin` |

### 4. Create the frontend service

```bash
railway add --service "frontend"
```

#### Connect GitHub repo (via API)

```bash
# Connect repo
curl -s -X POST https://backboard.railway.app/graphql/v2 \
  -H "Authorization: Bearer $RAILWAY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "mutation { serviceConnect(id: \"<FRONTEND_SERVICE_ID>\", input: { repo: \"sargy/gridpulse\", branch: \"feature/full-stack-rewrite\" }) { id } }"
  }'

# Set root directory and Dockerfile path
curl -s -X POST https://backboard.railway.app/graphql/v2 \
  -H "Authorization: Bearer $RAILWAY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "mutation { serviceInstanceUpdate(serviceId: \"<FRONTEND_SERVICE_ID>\", environmentId: \"<ENV_ID>\", input: { rootDirectory: \"/frontend\", dockerfilePath: \"Dockerfile\" }) }"
  }'
```

#### Set environment variables

```bash
railway service link frontend
railway variable set BACKEND_URL='http://${{backend.RAILWAY_PRIVATE_DOMAIN}}:8080'
railway variable set PORT=80
```

| Variable | Value | Notes |
|----------|-------|-------|
| `BACKEND_URL` | `http://${{backend.RAILWAY_PRIVATE_DOMAIN}}:8080` | Resolves to `http://backend.railway.internal:8080` |
| `PORT` | `80` | Port nginx listens on |

### 5. Generate public domains

```bash
railway service link frontend
railway domain
# Creates: https://frontend-production-XXXX.up.railway.app

railway service link backend
railway domain
# Creates: https://backend-production-XXXX.up.railway.app (optional — for direct API access)
```

### 6. Trigger deployment

Deployments happen automatically when:
- Code is pushed to the connected branch
- Environment variables are changed

Manual redeploy:
```bash
railway service link backend
railway redeploy -y

railway service link frontend
railway redeploy -y
```

## Secrets Location

All secrets are managed by Railway and never stored in git:

| Secret | Location | How to access |
|--------|----------|---------------|
| MongoDB password | Railway MongoDB service variables | `railway variable list` (link to MongoDB service) |
| MongoDB connection string | Backend service `SPRING_DATA_MONGODB_URI` | `railway service link backend && railway variable list` |
| Railway API token | `~/.railway/config.json` (local machine) | `railway whoami` to verify |

## Useful Commands

```bash
# Check project status
railway status

# View service logs
railway service link backend && railway logs -n 50
railway service link frontend && railway logs -n 50

# View build logs for latest deployment
railway logs --build --latest -n 100

# Open Railway dashboard
railway open
```

## Notes

- The Railway CLI `railway up` command uploads local files but does NOT respect `railway.toml` settings. For monorepo setups, use GitHub repo connections with `rootDirectory` set via the GraphQL API.
- The `railway.toml` files in `backend/` and `frontend/` are used by Railway when building from the connected GitHub repo.
- The frontend uses a custom `docker-entrypoint.sh` that runs `envsubst` to inject `BACKEND_URL` into the nginx config at container startup, avoiding substitution of nginx variables like `$host` and `$uri`.
